# Interactive PID Control Sandbox: Damped Motor Position

This repository contains an interactive, browser-based simulation designed for teaching introductory control systems. It allows students to tune a PID (Proportional-Integral-Derivative) controller and instantly visualize the effects on a rotational motor through both a static step-response analysis and a live 60-fps physics animation.

## 🚀 How to Run

There is no backend or build process required. The entire simulation runs locally in the browser.

1. Download or clone this repository.
2. Open `index.html` in any modern web browser.
3. **Alternatively, view it live here:** [https://mmax1406.github.io/pid-control-sandbox/]

## 🛠️ Features

* **Static Step Response Graph:** As you adjust the $K_p$, $K_i$, and $K_d$ gains, a Chart.js line graph instantly pre-calculates and displays the closed-loop step response (Position vs. Time) for a 1.0 radian target. 
* **Live Motor Animation:** A visual representation of the motor shaft. Enter a target angle in degrees and click "Execute Jump" to watch the physics engine animate the system in real-time.
* **Disturbance Injection:** Add a constant external torque to the system to test load-rejection and demonstrate the necessity of the Integral gain.
* **Live Metrics:** Automatically calculates Overshoot, Settling Time, and Steady-State Error.

## 📐 The Physics and Math

This simulation numerically integrates the physical equations of motion for a damped rotational mass. 

### 1. The Plant (System Dynamics)
The system models a rotational load (like a DC motor shaft) with inertia and viscous friction. The fundamental equation of motion is based on Newton's Second Law for rotation ($\sum \tau = J\alpha$):

$$J \ddot{\theta} + b \dot{\theta} = \tau_{total}$$

Where:
* **$J$** = Moment of Inertia (set to $1.0\ kg\cdot m^2$ in this simulation)
* **$b$** = Viscous Damping Coefficient (resists motion proportional to velocity)
* **$\theta$** = Current shaft angle (radians)
* **$\dot{\theta}$** = Angular velocity ($\omega$)
* **$\ddot{\theta}$** = Angular acceleration ($\alpha$)
* **$\tau_{total}$** = The net torque applied to the shaft

The total torque is the sum of the control effort from our PID controller and any external disturbances:

$$\tau_{total} = \tau_{pid} + \tau_{dist}$$

### 2. The PID Controller
The controller calculates the error $e(t)$ between the desired target angle ($\theta_{target}$) and the current angle ($\theta$):

$$e(t) = \theta_{target} - \theta(t)$$

The torque generated by the controller is:

$$\tau_{pid} = K_p e(t) + K_i \int e(t) dt - K_d \dot{\theta}$$

*Note on Derivative Kick:* Instead of taking the derivative of the error $\frac{d}{dt}e(t)$, this simulation calculates the derivative term using the negative velocity of the motor ($-K_d \dot{\theta}$). Because the target angle is a step function (which has an infinite derivative at $t=0$), using the derivative of the error would cause an massive, unrealistic spike in torque. Taking the derivative of the measurement is standard industry practice.

### 3. Numerical Integration
The simulation updates the physics in small time steps ($\Delta t = 0.01s$) using the Euler method:
1. Calculate Torque: $\tau = \text{PID Output} + \text{Disturbance}$
2. Calculate Acceleration: $\alpha = (\tau - b\cdot\omega) / J$
3. Update Velocity: $\omega_{new} = \omega_{old} + \alpha \cdot \Delta t$
4. Update Position: $\theta_{new} = \theta_{old} + \omega_{new} \cdot \Delta t$
